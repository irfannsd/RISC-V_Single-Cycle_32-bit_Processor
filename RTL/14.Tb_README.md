# RISC-V Single Cycle Processor – Testbench

## Overview

This testbench is used to verify the functionality of the **32-bit RISC-V Single-Cycle Processor** implemented in this repository.
It provides the necessary clock and reset signals, monitors processor execution, and prints the internal state of the CPU during simulation.

The testbench also generates a **VCD waveform file** that can be viewed in GTKWave to analyze signals such as the Program Counter (PC), instruction fetch, ALU results, register values, and memory operations.

The program stored in the instruction memory executes a small sequence of instructions to test:

* Arithmetic operations
* Immediate instructions
* Memory load/store
* Branch instruction
* Program termination

---

# Testbench Features

The testbench performs the following tasks:

### Clock Generation

A clock signal with a **10 ns period** is generated for the processor.

### Reset Initialization

The processor is reset at the beginning of the simulation to initialize all registers and memory states.

### Instruction Monitoring

The testbench continuously monitors important signals such as:

* Program Counter (PC)
* Current instruction
* ALU result
* Register values (x1 – x5)

These values are displayed in the terminal during simulation.

### Instruction Counter

The number of executed instructions is counted to verify program execution.

### Program Termination

The simulation stops when the processor encounters a **NOP instruction (`0x00000013`)**, which indicates the end of the program.

### Waveform Generation

A **VCD waveform file (`riscv.vcd`)** is generated for debugging and signal visualization.

# Instruction Execution Table

The following table shows the **instruction execution along with the terminal output values** during simulation.

| Time  | PC         | Machine Code | Assembly Instruction | Operation              | Register Result |
| ----- | ---------- | ------------ | -------------------- | ---------------------- | --------------- |
| 0     | `00000000` | `00000093`   | `addi x1, x0, 0`     | Initialize x1          | x1 = 0          |
| 25000 | `00000004` | `00500113`   | `addi x2, x0, 5`     | Load value 5 into x2   | x2 = 5          |
| 35000 | `00000008` | `002081b3`   | `add x3, x1, x2`     | x3 = x1 + x2           | x3 = 5          |
| 45000 | `0000000C` | `00310023`   | `sw x3, 0(x2)`       | Store x3 into memory   | mem[x2] = 5     |
| 55000 | `00000010` | `00010183`   | `lw x3, 0(x2)`       | Load value from memory | x3 = 5          |
| 65000 | `00000014` | `00118213`   | `addi x4, x3, 1`     | Increment x3 by 1      | x4 = 6          |
| 75000 | `00000018` | `0041a063`   | `blt x3, x4, offset` | Branch if x3 < x4      | 5 < 6 → True    |
| 85000 | `0000001C` | `00200293`   | `addi x5, x0, 2`     | Load constant value    | x5 = 2          |
| 95000 | `00000020` | `00000013`   | `nop`                | No operation           | Program ends    |

# Running the Simulation

Compile and run the testbench using **Icarus Verilog**:

```
iverilog -o riscv_sim 14.Testbench.v
vvp riscv_sim
```

Open the waveform file using **GTKWave**:

```
gtkwave riscv.vcd
```

# Purpose

This testbench verifies the correct functionality of the RISC-V single-cycle processor by testing instruction fetch, ALU operations, register writes, memory operations, and branch execution.
It serves as a debugging and validation tool for the processor design.
